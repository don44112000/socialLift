# Authentication

With the `auth_code` you receive after [authorization](https://business-api.tiktok.com/portal/docs?id=1738373141733378) by the advertiser, you can make a request to the following endpoint to get an `access_token` for subsequent API requests.


# Generate an access token
If you want to specify `Content-Type` as `application/json` to get a long-term access token, you can make a POST request to the `/oauth2/access_token/` endpoint. Specify `app_id`, `auth_code` and `secret` in the request. In the response, the list of advertiser IDs that this token can access, as well as the permission scope of this access token, will also be included.

The `auth_code` is valid for 1 hour and can be used only once. After the `auth_code` expires, you need to start over and perform the authorization steps again.

If you want to specify `Content-Type` as `application/x-www-form-urlencoded` to get a long-term access token, you need to call [/oauth/token/](https://ads.tiktok.com/marketing_api/docs?id=1739965703387137) instead.
> <span style="color:DodgerBlue">**Note**</span><br> If the access token is lost, please inform the advertiser to cancel authorization or contact the operation administrator.

## Comparing v1.2 and v1.3 
 The following table outlines the differences between v1.2 and v1.3 endpoints. 
 ```xtable
 |Changes{30%}|v1.2{35%}|v1.3{35%}|
|---|---|---|
|Endpoint path|v1.2/oauth2/access_token/|v1.3/oauth2/access_token/|
|Request parameter data type |`app_id `: number|`app_id`: string |
|Response parameter data type|`advertiser_ids`: number[]|`advertiser_ids`: string[]|
```

## Request 
**Endpoint** https://business-api.tiktok.com/open_api/v1.3/oauth2/access_token/

**Method** 
POST


**Header**
```xtable
|Field|Data Type|Description|
|---|---|---|
|Content-Type {Required}|string|Request message type <br>Accepted format: `"application/json"` |
```

**Parameters**
```xtable
|Field|Data Type|Description|
|---|---|---|
|app_id{Required}|string|ID of your developer app.<br><br> To find the <b>App ID</b>, navigate to [My Apps](https://ads.tiktok.com/marketing_api/apps/) > <b>App Detail</b> > <b>Basic Information</b>.|
|secret{Required}|string| Secret of your developer app. <br><br>To find the <b>Secret</b>, navigate to [My Apps](https://ads.tiktok.com/marketing_api/apps/) > <b>App Detail</b> > <b>Basic Information</b>. |
|auth_code {Required}|string|The authorization code (`auth_code`) generated by your <b>Advertiser authorization URL</b>. <br><br>The code is valid for one hour and can only be used once.<br><br>To find out how to get the authorization code based on the type of API you use, see the "<b>Authorization URL</b>" row in [Comparing authorization for different APIs](https://business-api.tiktok.com/portal/docs?id=1781891256601602#item-link-Comparing%20authorization%20for%20different%20APIs). |
```

### Example
```xcodeblock
(code curl http)
curl --location -request POST 'https://business-api.tiktok.com/open_api/v1.3/oauth2/access_token/' \
--header 'Content-Type: application/json' \
--data '{
    "app_id": "{{app_id}}",
    "secret": "{{secret}}",
    "auth_code": "{{auth_code}}"
}'
(/code)
```

# Invalidate an access token

A long-term access token does not expire, but it'll become invalid if the advertiser cancels the authorization. You can also use [/oauth2/revoke_token/](https://ads.tiktok.com/marketing_api/docs?id=1739965949088770) to invalidate an access token.

With the access token, you can access the endpoints according to the permissions that are granted by the advertiser, not the permissions that you apply for when creating your developer application.
 
To get a complete list of advertiser accounts that can be accessed with the access token, use the `/oauth2/advertiser/get/` endpoint.
