# Reporting

You can use Reporting API to run a report on your auction ads and reservation ads data, in synchronous mode or asynchronous mode. 
- To learn about an overview of Reporting API, see [Overview](https://ads.tiktok.com/marketing_api/docs?id=1751087777884161).  
- To learn about synchronous and asynchronous reports, and different report types,  see [Guides](https://ads.tiktok.com/marketing_api/docs?id=1751087792726017). 
  - [Synchronous and asynchronous reports](https://ads.tiktok.com/marketing_api/docs?id=1751088469160001): Introduces how to run synchronous and asynchronous reports.
  - [Report types](https://ads.tiktok.com/marketing_api/docs?id=1738864835805186): Introduces the **use cases** of basic reports, audience reports, playable ads reports, DSA reports, Business Center reports, and GMV max ads reports, and their supported dimensions, metrics and filters. 
  - [Best practices](https://ads.tiktok.com/marketing_api/docs?id=1738864817670145): Introduces the best practices to avoid or reduce time-outs in synchronous reports. 
  - [Data latency for reports](https://ads.tiktok.com/marketing_api/docs?id=1738864894606337): Introduces the data latency you might expect when using Reporting API. 
  - [FAQs](https://ads.tiktok.com/marketing_api/docs?id=1738864850353154): Introduces common questions about Reporting API. 
- To find out what APIs are available for running a report, see [API reference](https://ads.tiktok.com/marketing_api/docs?id=1740649343188994).
