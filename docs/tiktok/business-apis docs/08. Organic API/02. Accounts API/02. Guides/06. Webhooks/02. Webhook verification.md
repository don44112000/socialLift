# Webhook verification

This topic outlines how to conduct webhook verification. 

## Check the signature

TikTok webhooks are sent with a signature that the destination server can use to verify that the event came from TikTok and not a third party or malicious system.

It is <strong>strongly recommended</strong> that webhook consumers verify these signatures before processing each webhook event.

## Verify Message Signature

To protect your app against man-in-the-middle and replay attacks, you should verify the signature of messages sent to your application. Along with the webhook request body, the webhook delivery timestamp is also part of the signed payload, and so an attacker cannot change the timestamp without invalidating the signature. If the signature is valid but the timestamp is too old, you can have your application reject the payload.

The signature is included as the TikTok-Signature header in the webhook request.

### Example of TikTok-Signature
```xcodeblock
(code curl http)
"Tiktok-Signature": "t=1633174587,s=18494715036ac4416a1d0a673871a2edbcfc94d94bd88ccd2c5ec9b3425afe66"
(/code)
```

#### Signature Verification

**Step 1: Extract the timestamp and signatures from the header**

Split the `Tiktok-Signature` header, using the `,` character as the separator, to get a list of elements. Next, split each element, using the = character as the separator, to get a key and value pair.

The value for the key 't' corresponds to the **Unix timestamp in seconds**, and key ‘s’ corresponds to the **signature generated by the TikTok platform** upon webhook delivery.

**Step 2: Signature Generation**

The message to sign and compare with the received signature can be created by concatenating:
- The received timestamp as a string
- The character '.'
- A stringified JSON copy of the webhook request body

Once created, compute an HMAC using the SHA256 hash function, using your application secret as the key (provisioned for your application on the TikTok API For Business developer portal). Hex encode the generated HMAC for comparison with the received signature.

**Step 3: Signature Comparison**

Compare the signature in the header to the generated signature. In the case they are equal, compute the difference between the current Unix timestamp in seconds, and the received timestamp in the header. Use this to decide whether the difference is tolerable.

#### Sample Implementation

Node
```xcodeblock
(code curl http)
var CryptoJS = require("crypto-js");

// Example serialized copy of the webhook request to verify
const webhookRequest = {
  headers: {
    "host": "enj1gzfy7ltvrqf.m.pipedream.net",
    "x-amzn-trace-id": "Root=1-61de8b6a-1c7695005636528b40cac95f",
    "content-length": "251",
    "user-agent": "tiktok-webhook",
    "content-type": "application/json;charset=UTF-8",
    "tiktok-signature": "t=1641974633,s=dc023ece112b3a07d809896fd929928a132168d95d2a2e43aab0105fbb291df8",
    "webhook-schema-version": "v1.0",
    "x-tt-logid": "202201120803530102451591650189C78",
    "x-envoy-return-response-flags": "1",
    "rpc-persist-ori-path": "/"
  },
  body: {
    client_key: "7043848340043202562",
    event: "share.video.publish_complete",
    create_time: 1641974632,
    user_openid: "a81e31c6-3b57-40d7-9253-c2f211e70514",
    content: "{\"share_id\":\"v_publish.7052073813521844225\",\"video_id\":\"7044071604573310209\"}"
  }
};

// Replace your signing key (app secret) and acceptable webhook delay in seconds
const signingKey = "****************************************";
const acceptableDelay = 5;

// Collect the necessary information for webhook validation
const body = JSON.stringify(webhookRequest.body);
const keys = webhookRequest.headers["tiktok-signature"].split(',');
const receivedTimestamp = keys[0].split("=")[1];
const receivedSignature = keys[1].split("=")[1];

// Function to validate incoming webhook signature
function validateSignature (receivedTimestamp, receivedSignature, body) {
  // Create signed payload, generate HMAC, hex encode the HMAC
  const signedPayload = receivedTimestamp + '.' + body;
  const hmac = CryptoJS.HmacSHA256(signedPayload, signingKey);
  const computedSignature = CryptoJS.enc.Hex.stringify(hmac); 

  // If computed and received signatures match
  if (computedSignature === receivedSignature) {
    // Check if current timestamp within acceptable delay from received timestamp
    const currentTimestamp = Math.round(new Date().getTime() / 1000);
    const delay = currentTimestamp - receivedTimestamp;
    
    if (delay <= acceptableDelay) return true;
    else return false;
  }
  else return false;
};

if (validateSignature(receivedTimestamp, receivedSignature, body)) {
  // return a 200 HTTP response and continue with webhook processing
}
else {
  // return a 4xx HTTP response status and raise a verification exception
  // log the x-tt-logid header
}
(/code)
```
