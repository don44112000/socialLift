# Obtain a long-term access token

Use one of the two endpoints to generate a long-term access token: `/oauth2/access_token/` and `/oauth/token/`. 

A long-term access token is a type of token that you can obtain after receiving authorization from an ad account, a Business Center account, or a TikTok One Creator Marketplace account. The token does not expire. If you want to revoke a long-term access token, use [/oauth2/revoke_token/](https://business-api.tiktok.com/portal/docs?id=1739965949088770).

>  <span style="color:DodgerBlue">**Note**</span><br>
>  <strong>`/oauth2/access_token/` only supports `Content-Type` as `application/json`, while `/oauth/token/` supports `Content-Type` as `application/json` or `application/x-www-form-urlencoded`.</strong>

## /oauth2/access_token/
Use this endpoint to get a long-term access token that does not expire.

The only supported `Content-Type` value is `application/json`. To generate an access token via `Content-Type` as `application/x-www-form-urlencoded`, see the <strong>/oauth/token/</strong> section below.

### Comparing v1.2 and v1.3 
 The following table outlines the differences between v1.2 and v1.3 endpoints. 
 ```xtable
 |Changes{30%}|v1.2{35%}|v1.3{35%}|
|---|---|---|
|Endpoint path|/v1.2/oauth2/access_token/|/v1.3/oauth2/access_token/|
|Request parameter data type |`app_id `: number|`app_id`: string |
|New request parameters|/|`return_advertiser_ids`|
|Response parameter date type|`advertiser_ids`: number[]|`advertiser_ids`: string[]|
```

### Request 

**Endpoint** https://business-api.tiktok.com/open_api/v1.3/oauth2/access_token/

**Method** POST


**Header**
```xtable
|Field{30%}|Data Type{15%}|Description{55%}|
|---|---|---|
|Content-Type {Required}|string|Request message type <br>Accepted format: `"application/json"` |
```

**Parameters**
```xtable
|Field{30%}|Data Type{15%}|Description{55%}|
|---|---|---|
|app_id{Required}|string|ID of your developer app.<br><br> To find the <b>App ID</b>, navigate to [My Apps](https://ads.tiktok.com/marketing_api/apps/) > <b>App Detail</b> > <b>Basic Information</b>.|
|secret{Required}|string| Secret of your developer app. <br><br>To find the <b>Secret</b>, navigate to [My Apps](https://ads.tiktok.com/marketing_api/apps/) > <b>App Detail</b> > <b>Basic Information</b>. |
|auth_code {Required}|string|The authorization code (`auth_code`) generated by your <b>Advertiser authorization URL</b>. <br><br>The code is valid for one hour and can only be used once.<br><br>To find out how to get the authorization code based on the type of API you use, see the "<b>Authorization URL</b>" row in [Comparing authorization for different APIs](https://business-api.tiktok.com/portal/docs?id=1781891256601602#item-link-Comparing%20authorization%20for%20different%20APIs). |
|return_advertiser_ids|boolean|Whether to return the full list of ad accounts that the generated token can access through `advertiser_ids` in the response.<br><br>Supported values: `true`, `false`.<br><br>Default value: `true`.<br><br><p><span style="color:darkred"><b>Note</b></span>: If you have access to thousands of ad accounts or more, setting this field to `false` can prevent timeouts when retrieving the full list of all advertiser IDs in the response.</p>|
```

#### Example
```xcodeblock
(code curl http)
curl --location -request POST 'https://business-api.tiktok.com/open_api/v1.3/oauth2/access_token/' \
--header 'Content-Type: application/json' \
--data '{
    "app_id": "{{app_id}}",
    "secret": "{{secret}}",
    "auth_code": "{{auth_code}}"
}'
(/code)
```

```xtable
|message |string|Response message. For details, see [Return codes](https://ads.tiktok.com/marketing_api/docs?id=1737172488964097). |
|request_id |string| The unique ID of the request.|

#|scope |number[]| The scope of permissions granted to the access token. <br>See [Permission scope](https://ads.tiktok.com/marketing_api/docs?id=1753986142651394) to learn about the permission that a scope ID (value of `scope`) corresponds to.|
```

## /oauth/token/
Use this endpoint to get an access token. The supported `Content-Type` values are `application/x-www-form-urlencoded` and `application/json`. This is a new endpoint in TikTok API for Business v1.3.
 
### Request 
**Endpoint** https://business-api.tiktok.com/open_api/v1.3/oauth/token/

**Method** POST
 
**Header**
```xtable
|Field{30%}|Data Type{15%}|Description{55%}|
|---|---|---|
|Content-Type {Required}|string|Request message type <br>Accepted format: [`application/x-www-form-urlencoded`, `application/json`] |
```
 
**Parameters**
```xtable
|Field{30%}|Data Type{15%}|Description{55%}|
|---|---|---|
| client_id {Required} | string | ID of your developer app.<br><br> To find the <b>App ID</b>, navigate to [My Apps](https://ads.tiktok.com/marketing_api/apps/) > <b>App Detail</b> > <b>Basic Information</b>.|
| client_secret {Required} | string | Secret of your developer app. <br><br>To find the <b>Secret</b>, navigate to [My Apps](https://ads.tiktok.com/marketing_api/apps/) > <b>App Detail</b> > <b>Basic Information</b>. |
| code {Required} | string | The authorization code (`auth_code`) generated by your <b>Advertiser authorization URL</b>. <br><br>The code is valid for one hour and can only be used once.<br><br>To find out how to get the authorization code based on the type of API you use, see the "<b>Authorization URL</b>" row in [Comparing authorization for different APIs](https://business-api.tiktok.com/portal/docs?id=1781891256601602#item-link-Comparing%20authorization%20for%20different%20APIs). |
| grant_type {Required} | string | The way of generating the access token. <br><br> The allowed enum value: `authorization_code` (To generate the access token by authorization code). |
```
#### Example

##### `Content-Type` as `application/x-www-form-urlencoded`
```xcodeblock
(code curl http)
curl --location --request POST 'https://business-api.tiktok.com/open_api/v1.3/oauth/token/' \
--header 'Content-Type: application/x-www-form-urlencoded' \
--data-urlencode 'grant_type=authorization_code' \
--data-urlencode 'client_id={{client_id}}' \
--data-urlencode 'client_secret={{client_secret}}' \
--data-urlencode 'code={{code}}'
(/code)
```
 
##### `Content-Type` as `application/json`
```xcodeblock
(code curl http)
curl --location --request POST 'https://business-api.tiktok.com/open_api/v1.3/oauth/token/' \
--header 'Content-Type: application/json' \
--data-raw '{
    "grant_type": "authorization_code",
    "client_id": "{{client_id}}",
    "client_secret": "{{client_secret}}",
    "code": "{{code}}"
}'
(/code)
```
 
### Response
```xtable
|Field{30%}|Type{15%} |Description{55%}|
|---|---|---|
|code |number| Response code. For the complete list of response codes and descriptions, see [Return codes](https://ads.tiktok.com/marketing_api/docs?id=1737172488964097). |
|message |string| Response message. For details, see [Return codes](https://ads.tiktok.com/marketing_api/docs?id=1737172488964097). |
| request_id | string | The log ID of a request, which uniquely identifies the request. |
| access_token | string | The long-term access token for permission verification. |
| token_type | string | Token type. Enum values: `Bearer` (Bearer token. Anyone in possession of the token can use it to access the associated resources). |
| advertiser_ids | string[] | The list of ad accounts that the token can access. |
| scope | number[] | The endpoints that the token can access. <br>See [Permission scope](https://ads.tiktok.com/marketing_api/docs?id=1753986142651394) to learn about the permission that a scope ID (value of `scope`) corresponds to. |
```
