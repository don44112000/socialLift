# Events 1.0

> <span style="color:DodgerBlue">**Note**</span><br>
> The following six Events API 1.0 endpoints will be sunset in H2 2024. If you are still using these endpoints, we highly recommend that you upgrade to the Events API 2.0 as soon as possible.<ul><li>[/pixel/track/](https://ads.tiktok.com/marketing_api/docs?id=1740858531237890) and [/pixel/batch/](https://ads.tiktok.com/marketing_api/docs?id=1740858565852225)</li><li>[/offline/track/](https://ads.tiktok.com/marketing_api/docs?id=1758428013689857) and [/offline/batch/](https://ads.tiktok.com/marketing_api/docs?id=1758428053652482)</li><li>[/app/track/](https://ads.tiktok.com/marketing_api/docs?id=1740859196043266) and [/app/batch/](https://ads.tiktok.com/marketing_api/docs?id=1740859218541569)</li></ul><br><br>The Events API 2.0 ([/event/track/](https://ads.tiktok.com/marketing_api/docs?id=1771101303285761)) is now the recommended endpoint for reporting events from all sources, including web, app, offline and CRM (Customer relationship management) systems. This endpoint provides a unified API schema to help advertisers simplify and streamline their integration.

> <span style="color:DarkRed">**Important**</span><br>
> By using Events API endpoints, you agree to the [TikTok Business Products (Data) Terms](https://ads.tiktok.com/i18n/official/policy/business-products-terms) and that you will not share sensitive data with TikTok.

Events API consists of three endpoint groups: App Events, Pixel Events (Web Events), and Offline Events endpoints.
