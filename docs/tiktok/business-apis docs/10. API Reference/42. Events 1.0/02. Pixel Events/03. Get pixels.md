# Get pixels

Use this endpoint to obtain a list of Pixel information. The returned list contains [Pixel Event](https://ads.tiktok.com/help/article?aid=573722398920296903) list.
> <span style="color:DarkRed">**Important**</span><br>
> By using Events API endpoints, you agree to the [TikTok Business Products (Data) Terms](https://ads.tiktok.com/i18n/official/policy/business-products-terms) and that you will not share sensitive data with TikTok.
## Comparing v1.2 and v1.3 
 The following table outlines the differences between v1.2 and v1.3 endpoints. 
 ```xtable
|Changes{30%}|v1.2{35%}|v1.3{35%}|
|---|---|---|
|Endpoint path| /v1.2/pixel/list/| /v1.3/pixel/list/|
|Request parameter data type|`advertiser_id`: number <br>`pixel_id`: number |`advertiser_id`: string <br>`pixel_id`: string |
|New request parameters|/ | `filtering`<br>`available_for_catalog_only`|
|Response parameter name|`external_action` |`optimization_event` |
|Response parameter data type|`pixel_id`: number <br>`create_time`: number<br>`event_id`: number  |`pixel_id` : string <br>`create_time`: string  <br>`event_id`: string |
|Response parameter name and data type|`owned_by`: number |`owner_bc_id`: string |
|New response parameters|/ | `pixel_category`<br>`custom_event_type`<br> `ownership_status`<br>`automatic_advanced_matching_fields` <br>`enable_first_party_cookies`<br>`enable_expanded_data_sharing`|
```

## Request
**Endpoint** https://business-api.tiktok.com/open_api/v1.3/pixel/list/

**Method** GET

**Header**
```xtable
|Field{30%}|Data Type{15%}|Description{55%}|
|---|---|---|
|Access-Token {Required}|string|Authorized access token. For details, see [Authentication](https://ads.tiktok.com/marketing_api/docs?id=1738373164380162). |
```

**Parameters**
```xtable
|Field{30%}|Data Type{15%}|Description{55%}|
|---|---|---|
|advertiser_id {Required}|string| Advertiser ID |
|code |string| Pixel Code . If specified, the returned information will be filtered by Pixel <br>Code. |
|pixel_id |string| Pixel ID. If specified, the returned information will be filtered by Pixel ID. |
|name |string| Pixel name. Fuzzy matching is supported. If specified, the returned information will be filtered by pixel whose name contains the passed-in data. |
|order_by |string| Sorting method, optional values  include: `EARLIEST_CREATE` (Prioritizes the earliest created pixel), `LATEST_CREATE` (Prioritizes the latest created pixel). Default value is `EARLIEST_CREATE`|
|filtering|object|Filter conditions. |
#|available_for_catalog_only|boolean|Whether the pixel can be bound to a catalog as event source. Default value: `False`. To bind a pixel to a catalog, use [/catalog/eventsource/bind/](https://ads.tiktok.com/marketing_api/docs?id=1740492491200513).|
|page |number|Current page number. Default value is `1`, value range: ≥ 1|
|page_size |number|Page size. Default value is `10`, value range: 1-20|
```

### Example

```xcodeblock
(code curl http)
curl --location --request GET 'https://business-api.tiktok.com/open_api/v1.3/pixel/list/?advertiser_id={{advertiser_id}}&pixel_id={{pixel_id}}' \
--header 'Access-Token: {{Access-Token}}' \
(/code)
```

## Response
```xtable
|Field{35%}|Data Type{15%}|Description{50%}|

|code |number|Return code, see [Appendix-Return codes](https://ads.tiktok.com/marketing_api/docs?id=1737172488964097). |
|message |string|Return message, see [Appendix-Return information](https://ads.tiktok.com/marketing_api/docs?id=1737172488964097).|
|data |object|Return data. |
#|pixels|object[]| A list of Pixels.|
##|pixel_id |string| Pixel ID. |
##|pixel_code |string| Pixel Code. The ID you see on the Events Manager platform. |
##|pixel_name |string| Pixel name. |
##| pixel_category | string | The Pixel data connection event category for the Pixel. <br><br> Enum values: <ul><li>`ONLINE_STORE`: E-commerce events. </li><li> `FILLING_FORM`: Form. </li><li>`CONTACTS`: Consultation events. </li><li>`LANDING_PAGE`: APK download events.</li><li>`CUSTOMIZE_EVENTS`: Custom events.</li></ul><br><p><span style="color:darkred"><b>Note</b></span>: This field no longer determines the pixel events that the Pixel can measure and will be deprecated in the next API version.|
##|pixel_script |string| Pixel code, generated by the system. You need to include the Pixel code <br>on your website for event data connection. Please see [Help center](https://ads.tiktok.com/help/article?aid=6669727593823993861) for help.|
##|create_time |string| Time when the pixel was created, in the format of YYYY-MM-DD HH:MM:SS. See [ISO 8601](https://en.wikipedia.org/wiki/ISO_8601). |
##|pixel_setup_mode| string|Setup mode of the pixel. Enum values: `STANDARD`, `DEVELOPER`. Pixels that are created via the `/pixel/create/` endpoint after September 15th, 2022 are in the `DEVELOPER` mode.|
##|partner_name|string| Partner name.|
##|advanced_matching_fields|object|Manual Advanced Matching settings for the pixel.|
###|phone_number|boolean|Whether phone number is enabled for Manual Advanced Matching.<br><br>Supported values: `true`, `false`.|
###|email|boolean|Whether email is enabled for Manual Advanced Matching. <br><br>Supported values: `true`, `false`.|
##|automatic_advanced_matching_fields|object|Automatic Advanced Matching settings for the pixel.<br><br>Automatic Advanced Matching automatically uses customer information collected through your TikTok Pixel, like email, name, and phone number, to improve match rates, conversion tracking, and retargeting.|
###|phone_number|boolean|Whether phone number is enabled for Automatic Advanced Matching. <br><br>Supported values: `true`, `false`.|
###|email|boolean|Whether email is enabled for Automatic Advanced Matching. <br><br>Supported values: `true`, `false`.|
###|name|boolean|Whether name is enabled for Automatic Advanced Matching. <br><br>Supported values: `true`, `false`.|
###|address|boolean|Whether address is enabled for Automatic Advanced Matching. <br><br>Supported values: `true`, `false`.|
###|external_id|boolean|Whether external ID is enabled for Automatic Advanced Matching. <br><br>Supported values: `true`, `false`.|
##|enable_first_party_cookies|boolean|Whether to allow first-party cookies.<br><br>Allowing first-party cookies will allow the system to share data from your website's first-party cookies with TikTok to improve measurement and help reach more users. [Learn more about using cookies with TikTok Pixel](https://ads.tiktok.com/help/article/using-cookies-with-tiktok-pixel).<br><br>Supported values: `true`, `false`.|
##|enable_expanded_data_sharing|boolean|Whether to enable expanded data sharing.<br><br>Enabling expanded data sharing allows this pixel to collect more information from your website, including page content (such as meta tags, and product details), visitor actions (such as clicks, button interactions, and time spent), and website performance (such as page load speed and responsiveness). This data may be used to improve ad delivery and campaign performance on TikTok. You can view all collected data in TikTok Pixel Helper. [Learn more about Expanded Data Sharing with the TikTok Pixel](https://ads.tiktok.com/help/article/enhance-data-postback-with-the-tiktok-pixel).<br><br>Supported values: `true`, `false`.|
##|asset_ownership|object| Asset ownership data|
###| asset_relation_status | string | Asset relationship status. <br><br>Enum values: <ul><li> null: The pixel is owned by the advertiser (`advertiser_id`) and has not been transferred to a Business Center as an asset. </li><li>`TRANSFERRED`: The pixel has been transferred to a Business Center as an asset but is still owned by the advertiser (`advertiser_id`). To transfer a pixel, use [/bc/pixel/transfer/]( https://business-api.tiktok.com/portal/docs?id=1753994928261122). </li><li>`SHARED`: The pixel is shared with the advertiser (`advertiser_id`) as an asset. You can modify a pixel that is shared by others, but you cannot delete it.</li><li> `UNBOUND`: The pixel was shared with the advertiser (`advertiser_id`) as an asset but has been unbound from the advertiser. In reporting data, pixels whose status is `UNBOUND` will not be counted.</li></ul>
###| ownership_status | boolean | Whether the pixel is owned by the advertiser (`advertiser_id`). <br><br>Enum values: `true`, `false`. <br><br><ul><li> If the value of `asset_relation_status` is null or `TRANSFERRED`, the value of this field will be `true`. </li><li> If the value of `asset_relation_status` is `SHARED` or `UNBOUND`, the value of this field will be `false`. </li></ul>|
###| owner_bc_id | string | ID of the Business Center that owns the pixel as an asset. <br><br><ul><li> If `asset_relation_status` is `TRANSFERRED`, the value of this field represents the ID of the Business Center that the pixel has been transferred to as an asset. </li><li> If `asset_relation_status` is not `TRANSFERRED`, the value of this field will be null. </li></ul> |
###|updated_at{-To be deprecated}| integer|Time when the lastest relationship was formed, in the format of Unix timestamp (for example, 1581679723600). <br><p><span style="color:darkred"><b>Note</b></span>: The field is to be deprecated in the next API version, and currently you may not get a valid time via this field.|
##|events |object[]| A list of Pixel Events. <br>Pixel Events data is refreshed every 2-4 hours. If events occur within four hours, the value of this field might be an empty list (`[]`). You can retry after four hours to obtain the data of these events.|
###|currency_value |string| Event conversion value (if not specified when the event was created, it returns `"  "`). |
###|currency |string| Event corresponding currency. Enum values: `INR` (Indian Rupee), `JPY` (JPY), `USD`(USD). |
###|name |string| Event name. |
###|deprecated|boolean| Whether the event is deprecated. |
###|event_type |string| Returned only for a Standard Pixel Event.<br><br>Standard Pixel Event type.<br><br>To learn about the enum values, see the "<b>Event name for ad creation (`optimization_event`)</b>" column in [Supported Pixel events - Pixel events](https://business-api.tiktok.com/portal/docs?id=1739585696931842).|
###|optimization_event |string| The conversion event type for a Standard Pixel Event (`event_type`). You can pass the conversion event type to the `optimization_event` field when you create an ad group via [/adgroup/create/](https://business-api.tiktok.com/portal/docs?id=1739499616346114).<br><br>To learn about the enum values, see the "<b>Event name for ad creation (`optimization_event`)</b>" column in [Supported Pixel events - Pixel events](https://business-api.tiktok.com/portal/docs?id=1739585696931842).<br><br><p><span style="color:darkred"><b>Note</b></span>: The Standard Pixel Event type `PAGE_VIEW` cannot be used for optimization. Therefore, when `event_type` is `PAGE_VIEW`, this field will have a null value.  |
###| custom_event_type | string | Returned only for a Custom Pixel Event. <br><br>Custom Pixel Event type. Custom Events are actions that TikTok partners can define themselves beyond the predefined Standard Events list. To learn more about Custom Events, see [Standard and Custom Events](https://ads.tiktok.com/help/article/standard-events-parameters?lang=en). <br><br>You cannot create a Custom Pixel Event via [/pixel/event/create/](https://business-api.tiktok.com/portal/docs?id=1740858807646209), but you can send back a Custom Pixel Event directly by using the Events 1.0 endpoints [/pixel/track/](https://business-api.tiktok.com/portal/docs?id=1740858531237890) or [/pixel/batch/](https://business-api.tiktok.com/portal/docs?id=1740858565852225), or the Events 2.0 endpoint [/event/track/](https://business-api.tiktok.com/portal/docs?id=1771101303285761). For instance,  a `SHOPPING` event that is sent back will be regarded as a Custom Pixel Event because it's not included in the "<b>Event</b>" column in [Events API 1.0 - Report web events and parameters - Supported web events](https://business-api.tiktok.com/portal/docs?id=1741601162187777#item-link-Supported%20web%20events) or [Events API 2.0 - Supported events - Web Standard Events](https://business-api.tiktok.com/portal/docs?id=1771101186666498#item-link-Web%20Standard%20Events). |
###|event_code |string| Event Code. |
###|event_id |string| Event ID. |
###|statistic_type |string| Statistics type, optional values include: `EVERY_TIME` (every time),<br>`ONCE` (only once). |
###|rules |object[]| Data connection event rules, sub-field rules and descriptions see [Supported Pixel events-Event Rule Type](https://ads.tiktok.com/marketing_api/docs?id=1739585696931842). |
####|operator |string| Operator, optional values include: `OPERATORTYPE_CONTAINS`, `OPERATORTYPE_DOES_NOT_EQUAL`, `OPERATORTYPE_EQUALS`|
####|trigger |string| Event trigger type, optional values include: `TRIGGERTYPE_CLICK`, `TRIGGERTYPE_PAGEVIEW`.|
```
